Las preguntas que tengo que contestar:

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    Es muy bueno ya que el usuario puede introducir la información que le plazca... no necesariamente siempre va a ser válida. Aunque como buena páactica siempre se debe de validar los datos tanto en el frontend como en el backend

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    El usuario puede modificar el código de JavaScript en el navegador al seleccionar un elemento y quitarle sus eventos.

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    Es siempre bueno porque el usuario común no necesariamente tiene malas intenciones y/o sabe de tecnología. Lo más usual es que las personas cometan errores y lo mejor es anunciarles del error antes de hacer validaciones en el servidor.
