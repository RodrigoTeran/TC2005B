Las preguntas que tengo que contestar:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet es una red de computadoras, en cambio la World Wide Web es una red de páginas web.

  2. ¿Cuáles son las partes de un URL?

    Tiene 6 partes:

    1. Esquema: es el protocolo de la transmisión de datos. En la mayor parte de los casos el esquema se corresponde con: http o https.
    2. Host: normalmente, para el protocolo http, el host es: www.
    3. Dominio: corresponde con el nombre del dominio de cada sitio web, como: tec.mx
    4. Puerto: define el nombre del puerto según el host. Para el protocolo http, el puerto por defecto es 80, para https es 443.
    5. Directorio (o subdominio): define el camino que se debe seguir en el servidor. Por ejemplo: blog.
    6. Nombre de archivo: define más detalladamente las características del documento: videos-gratis.

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    Los métodos HTTP sirven para dar a entender que acción se necesita haecer a un recurso.

    Este es el propósito de cada uno:

    1. GET: Este verbo http solicita datos.
    2. HEAD: Este solicita datos pero no requiere de una respuesta en el objeto body. Se usa para pedir los headers de un recurso como si se hiciera una petición GET sin necesitar el body.
    3. POST: Este acción manda datos a un recurso. Esta acción causa que se cree un recurso en el servidor.
    4. PUT: Esta acción reemplaza todas las representaciones de un recurso por medio de los datos que se mandan.
    5. PATCH: Esta acción aplica una modificación parcial al servidor por medio de los datos que se mandan.
    6. DELETE: Esta acción elimina un recurso del servidor.
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    Se debe de usar el método: POST

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    Se debe de usar el método: GET

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    No, no ocurrió ningún error. Lo que paso fue que la petición fue procesada exitosamente.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    No es responsabilidad del desarrollador. Ya que un error 404 significa que un recurso solicitado no existe. Como una página web. Los errores en el rango de los 400 son errores del cliente. Entonces el desarrollador no tiene la responsabilidad de crear un recurso parcial (para que ahora sí exista).

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Claro, ya que un error 500 significa que hubo un error en el servidor. Los errores en el rango de los 500 son errores del lado del servidor (o se nuestro como desarrolladores). Es responsabilidad del desarrollador manejar los procedimientos del servidor para que el servidor se detenga por un proceso mal hecho.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Significa que los navegadores ya no soportan (no las leen correctamente). Entonces el lenguaje semántico que anteriormente proporcionaba estas etiquetas ya no van a servit.

    Elementos de HTML 4 que están desaprobados:

    • <marquee></marquee>
    • <center></center>
    • <applet></applet>
    • <big></big>

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    La diferencia más grande es que HTML 4 no tiene tantas etiquetas como HTML 5. Aunque HTML 5 no soporte algunas etiquetas de HTML 4, HTML 5 tiene muchas etiquetas nuevas y además es una versión más refiniada de HTML 4. Además, HTML 5 usualmente es llamado al conjunto de tecnologías de HTML, CSS y JavaScript.

  11. ¿Qué componentes de estructura y estilo tiene una tabla?

    Las tablas HTML están hechas de esta manera:

    Etiqueta Uso
    <table> Crea una tabla
    <th> Crea un header en la tabla
    <tr> Crea una fila
    <td> Crea una celda en la tabla
    <caption> Crea un título de la tabla
    <colgroup> Crea un grupo de columnas
    <col> Especifica propiedades de columna para cada columna dentro de un elemento <colgroup>
    <thead> Agrupa el contenido del encabezado en una tabla
    <tbody> Agrupa el contenido del cuerpo en una tabla
    <tfoot> Agrupa el contenido del footer en una tabla
  12. ¿Cuáles son los principales controles de una forma HTML5?

    Etiqueta Uso
    <form> Crea una forma
    <input> Crea una entrada de datos. Puede ser de muchos tipos: (texto, color, rango, etc)
    <label> Los <label> sirven para dar un mayor rango de un elemento al cual se está haciendo la referencia. También da un mejor SEO.
    <button> Crea un botón.
    <fieldset> Agrupa elementos en una forma
